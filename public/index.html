Pokedex API

API REST construida con NestJS y MongoDB

¿Qué es este proyecto?

Esta API proporciona acceso a una base de datos completa de Pokémon, permitiendo consultar, crear, actualizar y eliminar información sobre diferentes especies de Pokémon.

El proyecto está construido con NestJS, un framework progresivo de Node.js para la creación de aplicaciones del lado del servidor eficientes y escalables.

Endpoints disponibles

GET /api/v2/pokemon

Obtiene una lista paginada de todos los Pokémon disponibles.

Probar endpoint

GET /api/v2/pokemon/:term

Busca un Pokémon específico utilizando diferentes identificadores. El parámetro :term acepta:

  • Nombre del Pokémon (ej: pikachu)
  • ID de MongoDB (ej: 507f1f77bcf86cd799439011)
  • Número de la Pokédex (ej: 25)
Probar con "pikachu"

POST /api/v2/pokemon

Crea un nuevo Pokémon en la base de datos.

⚠️ Requiere un cuerpo JSON con la siguiente estructura:

{
  "name": "string",
  "no": "number"
}

Ejemplo:

{
  "name": "pikachu",
  "no": 25
}

PATCH /api/v2/pokemon/:term

Actualiza información de un Pokémon existente. El parámetro :term puede ser:

  • Nombre del Pokémon (ej: pikachu)
  • ID de MongoDB (ej: 507f1f77bcf86cd799439011)
  • Número de la Pokédex (ej: 25)

⚠️ Este endpoint requiere un cuerpo JSON con los datos a actualizar

DELETE /api/v2/pokemon/:term

Elimina un Pokémon de la base de datos. El parámetro :term acepta los mismos identificadores que el endpoint PATCH.

⚠️ Esta acción no se puede deshacer

Cargar datos iniciales

Si la base de datos está vacía, puedes cargarla con información de Pokémon usando el siguiente endpoint:

GET /api/v2/seed

Carga la base de datos con información de Pokémon desde la PokeAPI.

Ejecutar seed

⚠️ Ejecutar este endpoint más de una vez puede causar duplicados en la base de datos.

Tecnologías utilizadas

  • MongoDB
  • Mongoose
